    - **① 수집**: 전 종목 (Yahoo 스크리너 시총 $10B+ ∪ watchlist.txt 전수 — 지수 밖 핫종목·ADR·신규상장 커버)
    - **② 자격 (황금 등락률)**: M7 ±2% / 메가캡($200B+) ±3% / 그 외 ±4%
    - **③ 이유 검증 (엄격)**: 후보를 우선순위 순서로 철저히 조사 — 수집·조사·검토·팩트체크를 다해 반드시 이유를 찾으려 노력. 그럼에도 못 찾으면: |등락률| 7% 미만 → **제외** / 7% 이상 → 유지하되 **"상승 이유 미확인"(급등) / "하락 이유 미확인"(급락)** 표기. 제외로 빈 슬롯은 다음 후보로 채움(backfill). 지어내기 절대 금지
    - **④ 업종 동반 예외**: 개별 뉴스 없어도 동일 업종 동반 강세/약세면 포함, "업종 전반 강세/약세"로 설명. 같은 업종이 5개를 넘으면 대표 4개만 개별 게재하고 나머지는 **별도 묶음 행**("그 외 ○○ 동반 하락")으로 — 각 종목 %를 up/down 색과 함께 병기. **줄바꿈 처리: 종목명과 % 사이는 반드시 `&nbsp;`(예: `테라다인&nbsp;<span class="down">−13.6%</span>`)로 붙여 종목명·등락률이 갈라지지 않게 하고, 종목 나열 중간에 억지 `<br>`를 넣지 말 것(화면 폭에 맞춰 자동으로 흐름). `<br>`는 종목 나열과 '업종 공통 이유' 사이에 딱 하나만.** **대표 4개는 등락률순이 아니라 중요도 기준**(사건 당사자·메가캡·스토리 중심 — 예: D램 담합 소송일엔 당사자 마이크론은 등락률이 작아도 개별 게재)
    - **⑤ 선별**: 초과 시 우선순위 = M7·메가캡 → 뉴스 촉매 명확 → |등락률| 순으로 급등 10 · 급락 10
    - **⑥ 표시**: 급등은 상승률 큰 순, 급락은 하락폭 큰 순. **종목명은 반드시 한글**(티커 병기, 예: 샌디스크 (SNDK)). 태그·라벨 없이 종목명+등락률+촉매
    - 완결성 자가검증: qualified 목록과 페이지 대조, 큰 이동 누락 없는지 확인
    - 스크립트 실패 시(fallback): stockanalysis.com/markets/gainers·losers WebFetch로 대체
    - watchlist.txt는 핫섹터 변화에 따라 주기적으로 갱신 (새 핫테마 종목 추가)
  - 발표된 지표: **브리핑이 다루는 미국장 세션 '당일'에 실제 발표된 주요 지표를 빠짐없이** (예: 7/2 세션이면 7/2 발표분 전부, 다른 날 금지). "주요"=시장 반응 있는 것 전부(고용보고서 NFP·실업률·시간당임금, CPI·PCE·PPI, ISM, 소매판매, GDP, 신규실업수당청구, 공장주문/내구재 등) — **사소한 지표만 생략하고 주요 지표는 임의로 빼지 말 것**(매 실행 일관성: 같은 세션이면 같은 목록). 당일 발표가 없으면 표 대신 "· 당일 발표된 주요 지표 없음" 한 줄. **표 형식**(지표/결과/예상 3열). 결과 색: **예상 대비 시장에 긍정적 서프라이즈면 up(초록), 부정적이면 down(빨강), 부합·중립이면 무색** (예: 물가 예상 상회=빨강, 실업수당청구 예상 하회=초록)
  - 미국장 주요 일정(한국시간) — 섹션 제목도 이 이름 사용: **통합 표 1개(구분/내용/일시/비고)**. 구분 셀 색 태그: 지표=`tag s-ind`(파랑)·연준=`tag s-fed`(보라)·실적=`tag s-earn`(주황)·IPO=`tag s-ipo`(초록)·기타=`tag s-etc`(회색). **범위=오늘부터 1주일**, 날짜순 정렬, 오늘 일정은 일시에 "오늘(0/0)". **최대 8행**. 일시는 날짜 위·시간 아래 줄분리(`7/6 (월)<br>23:00`).
    - **★ 포함 우선순위 (명확한 기준 — 반드시 이 순서로 채운다)**:
      - **1순위 = 무조건 포함(in-window면 절대 누락 금지)**: ⓐ 핵심 지표 — **CPI·PCE·고용보고서(NFP)·FOMC(회의/의사록/의장 발언)·GDP·소매판매·ISM·PPI** ⓑ 대형·화제 IPO·상장, 특히 **한국 관련(예: SK하이닉스 ADR 상장)** ⓒ 어닝시즌 개막 대형은행. (FOMC·대형은행 개막·초대형 IPO는 1주 밖이어도 1~2행 "예고" 허용.)
      - **2순위 = 1순위 다 넣고 자리가 남을 때만, 중요도순**: 가늠자 실적(소비=리바이스·나이키·룰루레몬, 물류·항공=페덱스·델타, 반도체 장비 등 — 시총 작아도 화제면 포함, 비고에 역할 표기) → 메가캡 실적 → 기타 화제 이벤트.
    - **제한**: **실적 행은 최대 3~4개**(실적이 1순위 지표·IPO를 밀어내지 못하게 — 과거 실적 4개로 CPI·SK하이닉스 ADR이 밀려난 사례 있음). 같은 날 여러 실적이면 대표 1~2개만 + 비고 "그 외 다수".
    - **누락 금지(엄수)**: **8행 미만으로 자리가 남으면 1순위·자격 이벤트를 절대 빼지 말 것.** 형식(색태그 등)을 바꿔 재작성할 때도 기존 자격 행 삭제 금지. (실증 사례: CPI·SK하이닉스 ADR·리바이스가 6행짜리 표에 자리 있는데도 누락됨 — 재발 금지.)
    - **내용 없는 구분만 표 아래 텍스트 줄로**("IPO: 오늘 대형 IPO 없음" 식). 화제성 IPO의 상장 첫날 결과는 다음 날 특징주에도 반영.
    - **일시 셀 형식**: 날짜와 시간을 줄바꿈으로 분리 — `7/6 (월)<br>23:00` (날짜 위, 시간 아래). 시간 미정이면 날짜만.
  - **모바일 가독성(아이폰 기준 ~390px)**: 템플릿에 미디어쿼리 적용됨. 생성 시 텍스트 규칙 — 특징주 이유·키워드 설명은 **한 줄당 40자 이내**로 간결히, 뉴스 헤드라인은 35자 내외로 다듬기, '·'로 긴 나열을 만들지 않기(3개 이하 권장)

## 2) HTML 생성

- `template.html` 구조·CSS 그대로, 내용만 채워 `out/index.html` 로 저장.
- 색: 수치에만 (상승 초록 up / 하락 빨강 down). 소제목·강세/약세 단어는 회색.
- 한글 줄바꿈 `word-break:keep-all` (템플릿에 이미 적용).
- **키워드 설명은 키워드 아래 줄에** (kw-t 아래 kw-d, 인라인 " · 설명" 금지).
- **시장 지표 칩 6개, 2행×3열 그리드** (윗줄 10년물·달러인덱스·원/달러, 아랫줄 WTI·BTC·금 — 가로 스크롤 없음).
- **특징주 이유가 2개 이상이면 이유별로 `<br>` 줄바꿈** (긴 한 문장으로 어색하게 감싸지 말 것).

## 3) 최종 재검토(QA) — 게시 전 필수

**(A) 기계 검증 — `python3 verify.py` 를 반드시 실행한다.**
완성한 out/index.html의 지수 종가·등락률·특징주 등락률을 out/market.json·out/movers.json 원본과 자동 대조하고, 내부 정합성(등락폭↔등락률)·날짜·market.py 교차검증(야후 vs CNBC) 결과까지 확인한다. **"❌ 검증 실패"가 나오면 그 항목을 반드시 수정하고 다시 verify.py를 돌려 "✅ 검증 통과"가 될 때까지 반복**한다. 통과 전에는 게시 금지.

**(B) 사람 판단 검증 — 스크립트가 못 잡는 것:** 뉴스 5분야 채움/시간창/출처 실존/중복 없음 · 키워드 근거 · 특징주 촉매의 사실 여부(팩트체크) · 촉매를 못 찾은 종목의 미확인 표기 정확 · 날조 0. 문제 있으면 재수집·수정.
