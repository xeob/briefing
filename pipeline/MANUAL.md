## 1. 시스템 개요

### 1.1 목적
매일 아침 07:00 전까지, 한국 투자자 관점의 경제 정보(국내 뉴스·핫 이슈·직전 미국 세션 전체)를 **팩트 기반·오차 0·완결성·매일 일관된 구성**으로 담은 웹페이지를 자동 생성해 카카오톡으로 전달한다.

### 1.2 대원칙 (모든 규칙의 상위)
1. **신뢰가 생명** — 모든 수치·날짜·사실은 검증된 소스에서. 지어내기 절대 금지, 확인 불가는 "미확인" 표기.
2. **잘못된 브리핑보다 미발송이 낫다** — 기계 검증(verify.py) 통과 전엔 발송 불가.
3. **숫자·날짜는 스크립트가, 판단은 모델이** — 결정적 데이터는 코드로 수집·검증, 모델은 선별·서술·팩트체크만.
4. **소멸은 감사 가능하게** — 규칙상 브리핑에서 빠진 종목·정보는 실행 보고에 기록.
5. **매번 처음부터 새로** — 이전 브리핑 복사·재탕 금지.

### 1.3 아키텍처 흐름
```
[클라우드 루틴 06:30 KST, Fable 5]
  → repo 2개 클론 (xeob/briefing 공개 + xeob/briefing-secrets 비공개)
  → RUN.md 숙지
  → market.py / movers.py / events.py 실행  (결정적 데이터 3종)
  → 웹 리서치 (뉴스·키워드·섹터·재료·실적 — 2소스 교차)
  → template.html 구조로 out/index.html + out/summary.txt 생성
  → 사람판단 QA
  → ./publish.sh
       ├ verify.py 기계검사 9종 — 실패 시 여기서 중단(발송 불가)
       ├ GitHub push: index.html + archive/YYYY-MM-DD.html + (갱신된 events_static.json)
       └ 카카오 발송 (템플릿 134931, WDATE=그날 아카이브 링크) + 토큰 자동 갱신
  → 실행 보고 (URL·result_code·verify 결과·감사 기록)
```

## 5. 파트별 세부 매뉴얼

### 5.1 헤더
- `head-title` "경제 브리핑" · `head-date` "YYYY.MM.DD (요일)" · `head-sub` 고정 문구.
- 이모지 금지. 색은 수치에만(`.up` 초록/`.down` 빨강), 라벨·단어는 회색.

### 5.2 주요 뉴스
- **5분야 고정** 색태그: 정치 `t-pol`(빨강)·경제 `t-eco`(초록)·세계 `t-wor`(파랑)·IT `t-it`(보라)·산업 `t-ind`(주황). **분야당 3~5건** (2건 이하 금지).
- 5단계 선별: 수집→클러스터/중복제거→잡음 필터→점수화(영향범위·파급·신규성·보도밀도·시의성)→요약+검증.
- 헤드라인 35자 내외, 실존 원문 URL 링크. 경제 관련성 낮은 기사 지양.

### 5.3 핫한 키워드 Top 3
- 최근 24시간 급상승 경제 이슈. 소스: 구글 트렌드 급상승+포털 경제면 '많이 본 뉴스'+실시간 집계. 채점: 급등폭·중요도·교차등장·이유명확성.
- 형식: 순번(회색)+키워드(`kw-t`)+아래 줄 설명(`kw-d` 40자). 인라인 "· 설명" 금지.
- 한계: 공식 실검 부재 → 근사치 (유일한 스크립트 검증 밖 파트).

### 5.4 미국 시장 요약·해설
- `sum` 한 줄 + `cmt` 2~3줄. 직전 개장 세션 기준. 휴장 시 `sec-note` 명시.

### 5.5 지수 (4종, 2×2 카드 `idx-grid`)
- 다우 `^DJI`·나스닥 `^IXIC`·S&P 500 `^GSPC`·필라델피아 반도체 `^SOX` — 종가+등락폭+등락률.
- **값 = out/market.json 그대로.** Yahoo 일봉 종가 + CNBC(.DJI/.IXIC/.SPX/.SOX) 교차검증(0.2% 불일치·±10% 이상치 → errors 플래그 시에만 웹 재확인).

### 5.6 시장 지표 (6종, 2×3 칩 `chips` — 가로 스크롤 금지)
- 배열 고정: 윗줄 美10년물·달러인덱스·원/달러 / 아랫줄 WTI·BTC·금.
- **방안 A: 2소스 합의값 = 확정값.** CNBC realtime 1차 + Yahoo 교차 (investing과 소수 둘째 자리 미세차 감수).
- 심볼(Yahoo/CNBC): ^TNX/US10Y(**소수 3자리**) · DX-Y.NYB/.DXY · KRW=X/KRW= · CL=F/@CL.1 · BTC-USD/BTC.CM= · GC=F/@GC.1.

### 5.7 섹터
- 강세(강한 순)/약세(약한 순), `t-strong`/`t-weak` 태그 + 회색 `sector-note` 이유 40자.

### 5.8 특징주 — 4단계 알고리즘 (핵심)
**[1차 자격]** M7 ±2% / 메가캡($200B+) ±3% / 그 외 ±4% 전부 수집 (movers.json).
- M7 = 애플·MS·엔비디아·구글·아마존·메타·테슬라. 수집망 = 스크리너 top100($10B+) ∪ CORE_SCAN(M7+메가캡 ~58종 내장). 시총 전 종목 실측.

**[2차 재료 게이트]** (비메가캡 전원)
- **A급(확정 사실·1차 정보·공시)**: 실적/가이던스 발표·M&A 확정/공식 제안·규제/FDA/판결·대형 계약 공시(취소 포함)·파산/신용등급·지수 편입/제외·CEO 교체·내부자 매매·사고/리콜/해킹·감원·자사주/배당/증자·공매도 리포트. **본질 = ⓐ종목 자체의 구체적 사건 + ⓑ출처 확인 가능 → 예시 밖 유형도 A급.**
- **B급(2차 정보)**: 투자의견·목표가·애널 리포트·단독 보도/루머·간접 수혜/피해 보도.
- **C급(막연한 서술) = 재료 없음**: 업황 기대감·AI 수혜·차익실현·저가 매수·섹터 동반.
- 통과: A·B급. **$50B 이하는 A급만.** 재료 없으면 **|7%|+라도 개별 제외(예외 없음).**
- **M7·메가캡은 게이트 면제**(거시·섹터 사유 허용)하되 사유 조사 필수 — 출처 있는 사실만. 끝내 못 찾으면 이들만 "이유 미확인".

**[3차 배치]** 등락률순(동률 시총순) 상위 10 개별.
- **M7 = 10위 밖도 무조건 개별.**
- **메가캡 = 조건부**: 10위 밖이면 고유 재료 A/B급 보유 시에만 개별. 거시뿐이면 → 묶음 성립 시 묶음 / 아니면 섹터 파트·해설 + 감사 기록.
- 후보 10 미만이면 억지로 안 채움(6~8개 정상).

**[4차 동반 묶음]** (개별 이후 잔여만, 급등·급락 동일)
- 성립 = ⓐ같은 실제 업종에서 [메가캡 |3%|+ 또는 비메가캡 |5%|+] **5개 이상**(개별 포함 집계) AND ⓑ**묶음행에 남는 |5%|+ 3개 이상**(앵커 — verify 차단).
- 라벨 실제 업종명만(순환매·테마·낙폭과대·수혜 금지 — verify 차단). 멤버 = 티어 충족 잔여 전부(재료 없는 고% 종목 묶음 정당).

**[표기]** 한글 종목명(티커)·급등 상승률순·급락 하락폭순·이유 40자/줄(2개↑ `<br>`)·묶음행 종목명+% `&nbsp;` 결합·나열 중간 `<br>` 금지.

**[감사 기록]** (보고서만): ⓐ재료 없어 제외한 |7%|+ ⓑ10위 밖 잘린 재료 종목 ⓒ거시·단독 메가캡 — "제외: XYZ +9.2% 재료없음 / KO +3.5% 거시·단독".

**[종목 행선지 요약]**
| 상황 | 행선지 |
|---|---|
| 재료 있고 top10 | 개별 |
| M7 | 무조건 개별 |
| 메가캡·10위 밖·고유 재료 | 개별 |
| 메가캡·10위 밖·거시뿐 | 묶음(성립 시) → 섹터 파트+감사기록 |
| 비메가캡·재료 있음·10위 밖 | 묶음(성립 시) → 감사기록 |
| 재료 없음(등락률 불문) | 묶음(성립 시) → 완전 제외(7%+는 감사기록) |

### 5.10 미국장 주요 일정 (한국시간)
- **3열(구분/내용/일시) — 비고 없음.** 일시 한 줄 `7/14 (화) 21:30`(미정이면 날짜만). 날짜순, 오늘은 "오늘(0/0)". 구분 색태그 `s-ind`(파랑)/`s-fed`(보라)/`s-earn`(주황)/`s-ipo`(초록)/`s-etc`(회색).
- **연준 발언 = 발언자 이름** "연준 월러 발언"·"연준 의장 워시 발언" (events.py title 그대로). 내용 칸 짧은 꼬리표 허용("리바이스 (LEVI) — 소비 가늠자").
- **1순위(상한 없음·verify 차단)**: must_include 전부 + "시장(지수·환율·금리·유가·업종)을 움직이는가" 판단 — 핵심 지표(High 12)/정책·거시(관세·중앙은행·OPEC+·국채입찰·쿼드위칭·부채한도)/대형·한국 IPO/어닝 개막 은행/대형 M&A·규제. FOMC급은 1주 밖도 "예고".
- **2순위 기본 포함**: optional(Medium·$500M+ IPO). 넘칠 때만 최저우선 컷.
- **실적(상한 없음·개별 행)**: M7+메가캡+**가늠자 화이트리스트** — 소비: 리바이스·나이키·룰루레몬·스타벅스·맥도날드 / 물류·항공: 페덱스·UPS·델타·유나이티드 / 산업: 캐터필러·디어 / 반도체: 마이크론·ASML·TSMC / 미디어: 넷플릭스.

## 6. 데이터 스크립트 상세 + 출력 스키마

### 6.1 market.py → out/market.json
- 지수: Yahoo v8 chart 일봉 종가 배열(마지막 2개 확정봉) + CNBC restQuote 교차(0.2%/±10% 플래그). 지표: CNBC 1차 + Yahoo 교차(허용오차 10년물 0.03/DXY 0.2/원달러 3/WTI 0.4/BTC 500/금 12). curl 사용(로컬 urllib SSL 이슈).
- 스키마: `{generated_kst, indices:{이름:{close,chg,pct,verify}}, indicators:{이름:"표기값"}, errors:[], notes:[]}` — errors·notes 있으면 모델이 웹 재확인.

### 6.2 movers.py → out/movers.json
- 스크리너 day_gainers/losers 각 100($10B+) ∪ CORE_SCAN(±2% 사전필터). 시총: 스크리너→CNBC mktcapView(T/B/M)→MEGA_HINT.
- 스키마: `{generated_kst, rule, qualified_up:[...], qualified_down:[...], all:[{symbol,name,pct,mktcap_b,vol_ratio,src,tier,qualified}], errors}` — qualified 각 최대 25(M7·메가캡 전원+티어2 7%+ 전원 강제). 폴백: stockanalysis.com.

### 6.3 events.py → out/events.json
- Nasdaq 경제캘린더 오늘−3~+13일(재시도 3·간격 0.3s) + IPO($500M+) + events_manual.txt.
- 장치: FF 앵커 **오프셋 자동 캘리브레이션**(Nasdaq=ET+1일 실측) → ET→KST 변환(zoneinfo) → CANON 정규화(등록부·U-6 제외·예상치 보유 행 우선) → 연준 발언자 한글 이름(19명 매핑) → 중복 날짜 static 중재 → **static 자동 재갱신**(무오류일 때, 창 밖 보존 — publish가 push).
- 스키마: `{generated_kst, timezone, must_include:[{date(KST),time,title,impact,cat,src,keywords}], optional:[...], released:[{date(ET 세션일),kst,title,actual,forecast,previous}], errors, calibration, static_refresh}`.

### 6.4 events_manual.txt (KST 기입)
`YYYY-MM-DD | 구분 | 제목 | 키워드,쉼표 | 비고` — in-window(12일)면 must_include 강제(verify 차단 대상). 지난 날짜 자동 무시.
