## 1. 시스템 개요

### 1.1 목적
매일 아침 07:00 전까지, 한국 투자자 관점에서 필요한 경제 정보(국내 뉴스·핫 이슈·직전 미국 세션 전체)를 **팩트 기반·오차 0·완결성·매일 일관된 구성**으로 담은 웹페이지를 자동 생성해 카카오톡으로 전달한다.

### 1.2 대원칙 (모든 규칙의 상위)
1. **신뢰가 생명** — 모든 수치·날짜·사실은 검증된 소스에서. 지어내기 절대 금지, 확인 불가는 "미확인" 표기.
2. **잘못된 브리핑보다 미발송이 낫다** — 기계 검증(verify.py) 통과 전엔 발송 불가.
3. **숫자·날짜는 스크립트가, 판단은 모델이** — 결정적 데이터는 코드로 수집·검증, 모델은 선별·서술·팩트체크만.
4. **소멸은 감사 가능하게** — 규칙상 브리핑에서 빠진 종목·정보는 실행 보고에 기록(소리 없는 소멸 금지).
5. **매번 처음부터 새로** — 이전 브리핑 복사·재탕 금지.

### 1.3 아키텍처 흐름
```
[클라우드 루틴 06:30 KST, Fable 5]
  → repo 2개 클론 (xeob/briefing 공개 + xeob/briefing-secrets 비공개)
  → RUN.md 숙지
  → market.py / movers.py / events.py 실행  (결정적 데이터 3종)
  → 웹 리서치 (뉴스·키워드·섹터·재료·실적 — 2소스 교차)
  → template.html 구조로 out/index.html + out/summary.txt 생성
  → 사람판단 QA
  → ./publish.sh
       ├ verify.py 기계검사 9종 — 실패 시 여기서 중단(발송 불가)
       ├ GitHub push: index.html + archive/YYYY-MM-DD.html + (갱신된 events_static.json)
       └ 카카오 발송 (템플릿 134931, WDATE=그날 아카이브 링크) + 토큰 자동 갱신
  → 실행 보고 (URL·result_code·verify 결과·감사 기록)
```

## 3. 매 실행 필수 규칙 (과거 위반 이력으로 명문화 — 어기지 말 것)
1. **매번 처음부터 새로 생성.** 이전 브리핑(index.html·archive) 복사·재사용·부분수정 금지. 뉴스·키워드·특징주·일정 전부 오늘 새로 수집·작성.
2. **뉴스 시간창 엄수.** 각 기사의 게재시각을 확인. verify.py가 URL 날짜로 전전일 이전을 기계 차단, 15:20 시각 컷은 모델이 직접 확인.
3. **휴장/주말 처리.** 국내 뉴스·키워드는 시장 상태와 무관하게 매일 새로. 미국장 휴장 시 직전 개장 세션 종가 사용 + sec-note에 "○/○ 휴장 · 수치는 직전 거래일 ○/○ 종가" 명시. 미국 수치가 그대로라는 이유로 브리핑 전체 재탕 금지.
4. **template.html·RUN.md 세부규칙 전부 반영** + 마지막에 verify.py 통과 필수.

## 4. 실행 절차 (매일 8단계)
0. **비밀값 확보**: briefing-secrets/.env에서 6개 키 확보(값 출력 금지), `set -a; source ...; set +a`로 export. 없으면 중단 보고.
1. **RUN.md 전문 정독** (특히 필수 규칙·특징주 4단계).
2. **스크립트 3종 실행**: `python3 market.py` `python3 movers.py` `python3 events.py`. 실패 시에만 대체 소스(웹).
3. **웹 리서치**: 뉴스·키워드·섹터·특징주 재료·실적 일정 — 2소스 교차 확인, 시간창 엄수, 날짜 추측 금지(일정·발표지표는 events.json 값 사용).
4. **생성**: template.html 구조·CSS 그대로 out/index.html + out/summary.txt(카드용 요약 200자).
5. **사람판단 QA** (§7.2 체크리스트).
6. **./publish.sh** — verify 게이트 → 통과 시에만 게시+발송. 실패 시 지적 항목 수정 후 재실행 반복. 끝내 통과 못 하면 발송하지 않음.
7. **보고**: 게시 URL · 카카오 result_code · verify 결과 한 줄 + **감사 기록**(§5.8) + 실패 시 단계·사유.

무인 원칙: 사람이 안 보는 예약 실행 — 질문·선택지 제시하며 멈추지 말 것. 문제 시 스스로 최선 판단으로 계속(대체 소스·"미확인" 표기).

## 5. 파트별 세부 매뉴얼

### 5.1 헤더
- 구조: `head-title` "경제 브리핑" + `head-date` "YYYY.MM.DD (요일)" + `head-sub` 고정 문구.
- 섹션 제목·라벨에 이모지 금지. 색상은 수치에만(상승 초록 `.up` / 하락 빨강 `.down`), 라벨·단어는 회색.

### 5.2 주요 뉴스
- **5분야 고정** + 색 태그: 정치 `t-pol`(빨강)·경제 `t-eco`(초록)·세계 `t-wor`(파랑)·IT `t-it`(보라)·산업 `t-ind`(주황). **분야당 3~5건** (2건 이하 금지).
- 선별 5단계: 수집 → 클러스터/중복제거 → 잡음 필터 → 점수화(영향범위·파급·신규성·보도밀도·시의성) → 요약+검증.
- 형식: 헤드라인 35자 내외, 각 항목이 실존하는 원문 URL 링크(검정 텍스트+클릭). 시간창 §2.
- 경제 브리핑 관련성 낮은 기사(단순 정치 이벤트 등) 지양.

### 5.3 핫한 키워드 Top 3
- 기간: 실행 시점 기준 **최근 24시간** 급상승 경제 이슈.
- 소스 3종 종합: 구글 트렌드 급상승 + 포털 경제면 '많이 본 뉴스' + 실시간 이슈 집계. 채점: 급등폭·중요도·교차등장·이유명확성.
- 형식: 순번(회색) + 키워드(`kw-t`) + 아래 줄 배경 설명(`kw-d`, 40자 이내). 인라인 "· 설명" 금지.
- 알려진 한계: 한국은 2021년 이후 공식 실검 부재 → 근사치. 유일하게 스크립트 검증 밖 파트.

### 5.4 미국 시장 요약·해설
- `sum` 한 줄 요약 + `cmt` 2~3줄 원인 해설. 직전 개장 세션 기준.
- 휴장 시 `sec-note`: "7/3 휴장 · 수치는 직전 거래일 7/2(목) 종가" 형식으로 명시.

### 5.5 지수 (4종, 2×2 카드)
- 다우 `^DJI` · 나스닥 `^IXIC` · S&P 500 `^GSPC` · 필라델피아 반도체 `^SOX` — 종가+등락폭+등락률.
- **값 = out/market.json 그대로** (임의 수치·기사 속 수치 금지). market.py가 Yahoo 일봉 종가(휴일에도 정확) + **CNBC(.DJI/.IXIC/.SPX/.SOX) 교차검증** — 0.2% 초과 불일치·하루 ±10% 이상치는 errors 플래그 → 그때만 웹 재확인.

### 5.6 시장 지표 (6종, 2×3 칩 — 가로 스크롤 금지)
- 배열 고정: 윗줄 美10년물·달러인덱스·원/달러 / 아랫줄 WTI·BTC·금.
- 소스(방안 A — **2소스 합의값이 확정값**): CNBC realtime 1차 + Yahoo 교차. investing.com과 소수 둘째 자리 미세차는 감수(스크립트 접근 불가 소스라 강제 참조 아님).
- 심볼: 10년물 ^TNX/US10Y(**소수 3자리**, 예 4.485%) · 달러인덱스 DX-Y.NYB/.DXY · 원/달러 KRW=X/KRW= · WTI CL=F/@CL.1 · BTC BTC-USD/BTC.CM= · 금 GC=F/@GC.1.
- 야후 stale 감지 시 CNBC 자동 채택(notes 기록).

### 5.7 섹터
- 강세(강한 순)·약세(약한 순) 나열. 태그 `t-strong`(초록)/`t-weak`(빨강) + 회색 `sector-note` 이유 한 줄(40자).

### 5.8 특징주 — 4단계 알고리즘 (2026-07-04 최종 확정)

**[1차 자격]** movers.json qualified — **M7 ±2% / 메가캡($200B+) ±3% / 그 외 ±4%** 전부 수집.
- M7 = 애플·MS·엔비디아·구글·아마존·메타·테슬라.
- 수집망: Yahoo 스크리너 급등/급락 상위 100(시총 $10B+) ∪ **CORE_SCAN**(M7+주요 메가캡 ~58종, movers.py 내장 — 스크리너가 못 잡는 "M7 +2%대" 보장). 시총은 전 종목 실측(§6.2).

**[2차 재료 게이트]** (비메가캡 전원 적용)
- **재료 등급**:
  - **A급(확정 사실·1차 정보·공시)**: 실적/가이던스 발표, M&A 확정/공식 제안, 규제·FDA·판결 결과, 대형 계약·수주 공시(취소 포함), 파산·신용등급 변경, 지수 편입/제외, CEO 교체·내부자 매매, 사고·리콜·해킹, 감원·구조조정, 자사주/배당/증자 공시, 공매도 리포트 등. **본질 = ⓐ그 종목 자체의 구체적 사건 + ⓑ기사·공시 출처 확인 가능 — 충족하면 예시 밖 유형도 A급.**
  - **B급(2차 정보·의견)**: 투자의견·목표가 변경, 애널 리포트, 단독 보도/루머, 경쟁사 사건의 간접 수혜/피해 보도.
  - **C급(막연한 서술) = 재료 없음 취급**: "업황 기대감"·"AI 수혜"·"차익실현"·"저가 매수"·섹터 동반.
- **통과**: A·B급. **시총 $50B 이하는 A급만.** 재료 없으면 **|7%| 이상이라도 개별 제외(예외 없음)** — "정말 핵심 특징주만".
- **M7·메가캡은 게이트 면제**(후보 유지·거시/섹터 사유 서술 허용)하되 **사유 조사 필수** — 거시 사유도 출처 있는 사실 기반(예: "고용 쇼크 보도에 금리인하 기대"). 근거 없는 "시장 따라 상승" 금지. 끝내 못 찾으면 **M7·메가캡에 한해서만 "이유 미확인"** 표기(날조 방지 최후 폴백).

**[3차 배치]** 2차 통과 종목을 **등락률순(동률은 시총순) 상위 10개** 개별 게재.
- **예외① M7 = 10위 밖이어도 무조건 개별** (시장 그 자체, 최대 +7행 감수).
- **예외② 메가캡 = 조건부**: 10위 밖이면 **고유(종목 자체) 재료 A/B급 보유 시에만** 개별(예: D램 담합 당사자 마이크론). 거시·섹터 사유뿐이면 개별 제외 → 묶음 성립 시 묶음행으로(% 보존), 아니면 섹터 파트·시장 해설이 커버 + 감사 기록. (근거: 무조건 예외였다면 7/2 혼조세에도 개별 34행 — 거시 사유 도배 잡음.)
- 후보 10 미만이면 **억지로 채우지 않음** (6~8개도 정상).

**[4차 동반 묶음]** (개별 배치 이후 잔여만)
- **성립 조건(둘 다 충족)**: 같은 '실제 업종'에서
  - ⓐ **[메가캡 |3%|+ 또는 비메가캡 |5%|+] 종목 5개 이상** (개별 게재분 포함 집계)
  - ⓑ **묶음행에 실제로 남는 |5%|+ 종목 3개 이상** (앵커 — verify.py가 기계 차단)
- 라벨 = **실제 업종명만** (반도체·은행·방산·항공·에너지·헬스케어…). **순환매·낙폭과대·테마·수혜 라벨 금지**(verify 금지어 차단).
- 멤버 = 티어 충족 업종 동반 종목 중 개별 제외 잔여 전부 — **재료 없어 개별에서 빠진 고등락률(7%+ 포함) 종목도 묶음엔 정당**(업종 동반이 그 종목의 이유).
- 급등·급락 동일 적용. 조건 미달이면 묶음 없이 개별만.

**[표기]** 종목명 반드시 한글+티커 병기(예: 샌디스크 (SNDK)). 급등 상승률순·급락 하락폭순. 이유 40자/줄, 2개 이상이면 `<br>`. 묶음행은 종목명과 % 사이 `&nbsp;` 결합(줄바꿈에 이름·%가 갈라지지 않게), 나열 중간 억지 `<br>` 금지, `<br>`는 나열↔공통 이유 사이 딱 하나.

**[감사 기록]** (브리핑 미게재, 실행 보고에만): ⓐ재료 없어 개별 제외된 |7%|+ ⓑ등락률 10위 밖으로 잘린 재료 보유 종목 ⓒ거시 사유뿐이라 개별·묶음 모두 못 실린 메가캡 — "제외: XYZ +9.2% 재료없음 / KO +3.5% 거시·단독" 형식.

**[종목의 행선지 요약]**
| 상황 | 행선지 |
|---|---|
| 재료 있고 top10 | 개별 |
| M7 | 무조건 개별 |
| 메가캡, 10위 밖 + 고유 재료 | 개별 |
| 메가캡, 10위 밖 + 거시뿐 | 묶음(성립 시) → 아니면 섹터 파트+감사기록 |
| 비메가캡 재료 있음, 10위 밖 | 묶음(업종 동반+성립 시) → 아니면 감사기록 |
| 재료 없음(등락률 불문) | 묶음(업종 동반+성립 시) → 아니면 완전 제외(+7%+는 감사기록) |

### 5.10 미국장 주요 일정 (한국시간)
- **형식**: 통합 표 1개, **3열(구분/내용/일시) — 비고 칸 없음**. 날짜순, 오늘 일정은 "오늘(0/0)". **일시 = 날짜·요일·시간 한 줄** `7/14 (화) 21:30` (시간 미정이면 날짜만). 구분 색태그: 지표 `s-ind`(파랑)·연준 `s-fed`(보라)·실적 `s-earn`(주황)·IPO `s-ipo`(초록)·기타 `s-etc`(회색).
- **연준 발언은 발언자 이름 표기**: "연준 월러 발언"·"연준 의장 워시 발언" (events.py가 자동 생성 — 임의 표기 금지).
- 내용 칸엔 짧은 꼬리표만 허용: "JP모건 등 대형은행 — 어닝 개막", "리바이스 (LEVI) — 소비 가늠자".
- **포함 우선순위**:
  - **1순위(상한 없음·절대 누락 금지)** = events.json `must_include` 전부(verify가 누락 차단) + 판단 기준 **"시장(지수·환율·금리·유가·업종)을 움직일 수 있는가 — 그렇다면 목록 밖도 포함"**: 핵심 지표(High 12) / 정책·거시(관세·무역협상, 주요국 중앙은행, OPEC+, 국채 입찰, 쿼드러플 위칭, 부채한도·셧다운) / 대형·한국 IPO(예: SK하이닉스 ADR) / 어닝 개막 대형은행 / 대형 M&A·규제 결정. FOMC·대형은행 개막·초대형 IPO는 1주 밖이어도 "예고" 허용.
  - **2순위(기본 포함)** = events.json `optional`(Medium 지표·$500M+ IPO). 표가 길어져도 포함, 정말 넘칠 때만 최저우선(중복성 연준 발언 등)만 컷.
  - **실적(별도 트랙·상한 없음·개별 행)** = ①M7 ②메가캡($200B+) ③**경기 가늠자 화이트리스트**: 소비=리바이스·나이키·룰루레몬·스타벅스·맥도날드 / 물류·항공=페덱스·UPS·델타·유나이티드 / 산업=캐터필러·디어 / 반도체(한국 관심)=마이크론·ASML·TSMC / 미디어=넷플릭스. 몰려도 "그 외 다수"로 뭉개지 말 것. 화이트리스트 밖 소형 실적만 제외.
- **행 수**: 목표 8~12행. **1순위·대상 실적은 12행 초과해서라도 전부** — 정보손실 최소화가 행 수보다 우선. 재작성·형식 변경 시에도 기존 자격 행 삭제 금지.

### 6.3 events.py → out/events.json
- 수집: Nasdaq 경제캘린더 **오늘−3 ~ +13일** 날짜별 조회(재시도 3회·백오프·호출 간 0.3s) + Nasdaq IPO($500M+) + events_manual.txt.
- **날짜 캘리브레이션**: Nasdaq 날짜는 실제 ET+1일 어긋남(앵커 5건 실측) → 매 실행 Forex Factory(공식 ET 날짜) 앵커와 대조해 오프셋 자동 산출(폴백 +1).

- **중복 날짜 중재**: 같은 지표가 인접 날짜 양쪽에 잡히면(소스 노이즈) 신선한(≤14일) static 날짜 기준으로 정리.
- **정적 폴백**: 라이브 조회가 빠뜨린 1순위를 events_static.json으로 보충(+경고). **자동 재갱신**: 라이브 조회 무오류인 날 static을 오늘 데이터로 재생성(창 밖 이벤트 보존) → publish.sh가 repo에 push → 다음 날 클론에 최신본. 14일 경과 시 경고(비상시에만 gen_static.py 수동).
- 출력: must_include(1순위) / optional(2순위) / released(발표분, actual·forecast) / errors / calibration.

## 8. 게시·발송 — publish.sh (단일 경로, 다른 방법 금지)
1. **[게이트]** `python3 verify.py` — 실패(exit 1) 시 게시·발송 자동 중단.
2. **GitHub 게시**: index.html 교체 + `archive/YYYY-MM-DD.html`(KST 날짜) 추가 + 갱신된 events_static.json 동반 → main push → Pages 1~2분 내 갱신.
3. **카카오 발송**: refresh_token → access_token → `talk/memo/send` 커스텀 템플릿 **134931**, `template_args = {"DATE":"M/D", "SUMMARY":summary.txt(200자), "WDATE":"YYYY-MM-DD"}`. 성공 = `{"result_code":0}`.
   - **WDATE**: 카카오 콘솔의 링크 Path가 `/briefing/archive/${WDATE}.html` → **카드마다 그날의 영구 아카이브로 고정** (과거 카드가 최신으로 안 바뀜).
   - **토큰 슬라이딩 자동 갱신**: 카카오는 만료 1개월 전부터 갱신 호출에 새 refresh_token 발급 → publish.sh가 수신 즉시 .env/briefing-secrets 자동 교체+push → **정상 운영 중 수동 재인증 영구 불필요**.
